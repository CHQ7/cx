# CX 工作流程

遵循 **设计 → 开发 → 验证 → 反思 → 改进** 的循环迭代。

## 各阶段说明

### 1. 设计阶段

**输出**：架构设计文档 → `docs/superpowers/specs/`

- 系统结构
- 接口定义
- 数据模型
- 验收标准

### 2. 开发阶段

**输出**：开发计划 → `docs/superpowers/plans/`

- 任务分解
- 实现步骤
- 验证方法

**开发后总结**：每次开发完成后，在 `docs/superpowers/iterations/` 创建迭代记录

### 3. 验证阶段

**输出**：验证记录 → `docs/superpowers/reviews/`

- 测试用例执行结果
- 代码审查记录

### 4. 反思阶段

**输出**：迭代总结 → `docs/superpowers/iterations/`

- 完成情况
- 遇到的问题
- 解决方案
- 经验教训

### 5. 改进阶段

**输出**：改进计划 → `docs/superpowers/improvements/`

- 优化方案
- 循环回设计阶段

## 文档命名规范

```
specs/YYYY-MM-DD-<name>-design.md
plans/YYYY-MM-DD-<name>.md
reviews/YYYY-MM-DD-<name>-review.md
iterations/YYYY-MM-DD-<iteration>-summary.md
improvements/YYYY-MM-DD-<improvement>.md
```

## 迭代记录模板

每次开发完成后，创建迭代记录文档：

```markdown
# YYYY-MM-DD 迭代总结

## 关联项目

- 代码仓库：`<path>`
- 开发计划：`[[YYYY-MM-DD-<name>]]`

## 完成内容

- [x] 任务1
- [x] 任务2

## 遇到的问题

1. 问题描述 → 解决方案

## 经验教训

- ...

## 下一步计划

- ...
```
