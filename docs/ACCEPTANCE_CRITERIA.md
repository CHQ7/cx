# cx 分阶段验收标准（TUI 最终阶段）

本文档定义 `cx` 项目从 MVP 到完整交付的五个阶段的验收标准。**TUI 界面推迟到第五阶段**，前四个阶段均以 CLI（`cx run`）方式验证。

每个阶段完成后，必须通过所有验收项，方可进入下一阶段。

---

## 第一阶段：MVP 核心（纯 CLI）

**目标**：Agent 循环 + 文件读写/替换工具 + OpenAI 适配 + `cx run` 命令。

### 验收项

| # | 验收项 | 验证方式 | 状态 |
|---|--------|----------|------|
| 1.1 | `cargo build --release` 无错误，生成二进制 `cx` | 运行命令 | ☐ |
| 1.2 | 执行 `cx run "创建一个 test.txt，内容为 hello world"`，文件创建且内容正确 | 检查文件系统 | ☐ |
| 1.3 | 执行 `cx run "读取 test.txt"`，终端输出文件内容 | 观察输出 | ☐ |
| 1.4 | 执行 `cx run "把 test.txt 中的 world 替换为 cx"`，文件内容变为 “hello cx” | 检查文件内容 | ☐ |
| 1.5 | 配置 `~/.cx/config.toml`（api_key, model），运行任务时读取配置并生效 | 修改配置后运行任务 | ☐ |
| 1.6 | Agent 循环正确处理无工具调用的普通对话（如 `cx run "写一首诗"`），输出 LLM 回复 | 观察输出 | ☐ |
| 1.7 | 错误处理：LLM 返回无效 JSON、工具执行失败时，程序不崩溃，输出友好错误 | 模拟错误（如错误 API Key） | ☐ |
| 1.8 | 空闲时内存占用 < 50 MB，单次任务端到端延迟不超过 LLM API 延迟 + 500ms | `time` + `htop` | ☐ |

> 第一阶段不要求：会话持久化、技能固化、除文件工具外的其他工具、TUI。

---

## 第二阶段：工具扩展与会话持久化

**目标**：增加 Shell、网页扫描、用户询问工具；L1 会话检查点可恢复；支持任务队列（CLI 排队）。

### 验收项

| # | 验收项 | 验证方式 | 状态 |
|---|--------|----------|------|
| 2.1 | `cx run "列出当前目录文件"` 应正确执行 `ls`/`dir` 并输出结果 | `code_run` 工具，观察输出 | ☐ |
| 2.2 | `cx run "扫描 https://example.com 并提取页面标题"` 返回正确标题 | `web_scan` 工具 | ☐ |
| 2.3 | Agent 调用 `ask_user` 时，程序暂停并提示用户输入，用户输入后继续执行 | 设计一个需要确认的任务（如“删除前确认”） | ☐ |
| 2.4 | 执行 `cx run "长任务"` 中途按 Ctrl+C，再次运行 `cx resume`（或自动加载检查点）可从中断处继续 | 使用 `update_working_checkpoint` 保存状态 | ☐ |
| 2.5 | 同时提交两个耗时任务（如 `sleep 5` 和 `sleep 3`），第二个任务在前一个完成后自动开始 | 使用 `cx run --queue` 或顺序调用（依赖任务队列） | ☐ |
| 2.6 | 所有工具调用均有超时控制（默认 30 秒），超时后返回错误，Agent 循环继续 | 执行 `cx run "sleep 60"`（设置超时 5 秒） | ☐ |

## 第三阶段：完整记忆与技能固化

**目标**：实现 L0/L2/L3 记忆；技能自动生成、存储与检索（CLI 方式查看技能）。

### 验收项

| # | 验收项 | 验证方式 | 状态 |
|---|--------|----------|------|
| 3.1 | 修改 `~/.cx/soul/SOUL.md` 添加“你总是用海盗口吻说话”，`cx run "介绍一下自己"` 回复风格改变 | 对话测试 | ☐ |
| 3.2 | 在 `~/.cx/user/USER.md` 设置 `默认工作目录=/tmp`，`cx run "创建一个 a.txt"` 文件应在 `/tmp` 下创建 | 检查文件位置 | ☐ |
| 3.3 | 手动创建 `~/.cx/skills/test_skill/SKILL.md`（含 `triggers: ["测试技能"]` 和步骤），`cx run "测试技能"` 应直接执行该技能且不产生 LLM 工具调用（或极快执行） | 观察日志，应显示“使用技能 xxx” | ☐ |
| 3.4 | 执行多步骤任务（例如“下载 https://example.com 并提取所有链接”），任务成功后 `skills/` 下自动生成一个新的 `SKILL.md` | 检查文件系统，内容应包含步骤 | ☐ |
| 3.5 | 再次执行类似任务（如“下载 https://another.com 并提取链接”），Agent 直接调用已生成的技能，无需再次 LLM 规划 | 比较两次执行时间（第二次应明显更快） | ☐ |
| 3.6 | `cx skill list` 命令列出所有已安装技能及其描述 | 运行命令，检查输出 | ☐ |

## 第四阶段：高级工具与跨平台

**目标**：浏览器 CDP 控制、ADB 控制、屏幕截图 + 键鼠模拟；三平台测试通过（仍为 CLI 验证）。

### 验收项

| # | 验收项 | 验证方式 | 状态 |
|---|--------|----------|------|
| 4.1 | `cx run "打开浏览器访问 github.com 并搜索 cx"`，真实/headless 浏览器完成操作 | 需要本机安装 Chrome/Chromium | ☐ |
| 4.2 | `cx run "在当前页面执行 document.title 并返回"` 返回正确标题 | `web_execute_js` 工具 | ☐ |
| 4.3 | 连接 Android 设备（ADB 调试），`cx run "截图并保存到 /tmp/screen.png"` 生成文件 | 检查文件 | ☐ |
| 4.4 | `cx run "获取屏幕截图并保存到 /tmp/capture.png"` 生成截图文件 | `xcap` 工具 | ☐ |
| 4.5 | `cx run "模拟键盘输入 'Hello cx' 后按回车"` 在文本编辑器中观察到输入（谨慎测试） | 手动验证 | ☐ |
| 4.6 | 在 **Windows 10/11**、**macOS 14+**、**Ubuntu 22.04** 上分别编译并运行所有高级工具（浏览器/ADB/键鼠需预装依赖，但 `cx` 本身不 panic） | 三平台测试 | ☐ |

> 注意：4.5 可能干扰用户，建议在专用测试环境或使用虚拟显示器。

## 第五阶段：终端界面打磨

**目标**：实现完整的 `ratatui` TUI，替换 CLI 交互模式，提供可视化任务管理、技能面板、实时日志。

### 验收项

| # | 验收项 | 验证方式 | 状态 |
|---|--------|----------|------|
| 5.1 | 无参数运行 `cx` 进入 TUI，显示对话区域、输入框、命令面板、技能列表 | 手动启动 | ☐ |
| 5.2 | TUI 中可输入自然语言任务，Agent 执行并实时流式显示 LLM 思考过程（token by token） | 输入任务，观察逐字输出 | ☐ |
| 5.3 | 支持 `/` 开头的命令：`/help`、`/skill list`、`/model set`、`/stop` | 输入各命令，验证功能 | ☐ |
| 5.4 | 当 Agent 调用 `ask_user` 时，TUI 弹出模态输入框，用户输入后任务继续 | 设计需确认的任务 | ☐ |
| 5.5 | TUI 中可查看当前会话的历史消息（滚动），支持分页 | 产生超过一屏的消息，滚动查看 | ☐ |
| 5.6 | 技能固化时，TUI 右下角技能列表自动刷新，显示新技能 | 执行一个可固化的任务，观察技能列表更新 | ☐ |
| 5.7 | 按 Ctrl+C 优雅退出 TUI，无残留子进程 | 启动 TUI，执行任务中途退出，检查进程 | ☐ |
| 5.8 | 空闲时 TUI 内存占用 < 60 MB，界面刷新流畅（≥30 FPS） | `htop` + 视觉感受 | ☐ |
